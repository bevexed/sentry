# Sentry CSRF 智能配置说明

## 配置方案

当前使用 **方案 4：智能通配符配置**，无需每次修改 IP 地址。

## 工作原理

`sentry.conf.py` 中的智能配置会根据你在 `.env` 中设置的 `SENTRY_URL_PREFIX` 自动添加 CSRF 可信源：

### 1. 基础配置
无论设置什么 URL，都会自动包含：
- 你配置的 URL
- `http://localhost:9000`
- `http://127.0.0.1:9000`

### 2. 内网 IP 自动扩展

#### 192.168.x.x 网段
如果你设置 `SENTRY_URL_PREFIX=http://192.168.158.244:9000`
- 自动允许 `http://192.168.158.1:9000` ~ `http://192.168.158.254:9000`
- 即整个 `192.168.158.x` C 段的所有 IP

#### 10.x.x.x 网段
如果你设置 `SENTRY_URL_PREFIX=http://10.0.1.100:9000`
- 自动允许 `http://10.0.1.1:9000` ~ `http://10.0.1.254:9000`

#### 172.16-31.x.x 网段
如果你设置 `SENTRY_URL_PREFIX=http://172.16.0.100:9000`
- 自动允许 `http://172.16.0.1:9000` ~ `http://172.16.0.254:9000`

## 使用场景

### 场景 1：IP 地址经常变化
如果你的电脑 IP 在 `192.168.158.x` 网段内经常变化：
- 只需设置一次 `SENTRY_URL_PREFIX=http://192.168.158.244:9000`
- 即使 IP 变成 `192.168.158.100`，也能正常访问，无需修改配置

### 场景 2：多人访问
团队成员在同一内网（如 `192.168.1.x`）访问 Sentry：
- 只需设置 `SENTRY_URL_PREFIX=http://192.168.1.100:9000`
- 所有 `192.168.1.1-254` 的设备都能访问

### 场景 3：开发环境
本地开发测试：
- 设置 `SENTRY_URL_PREFIX=http://localhost:9000`
- 可以通过 localhost 和 127.0.0.1 访问

## 配置文件

### .env
```bash
SENTRY_URL_PREFIX=http://192.168.158.244:9000
```

### sentry.conf.py
智能配置代码会自动处理，无需手动修改。

## 优势

✅ **一次配置，永久有效** - 同一网段 IP 变化无需重新配置  
✅ **支持多人访问** - 团队成员无需各自配置  
✅ **自动识别网段** - 支持常见的内网 IP 段  
✅ **安全可控** - 只允许同一 C 段，不会过度开放  

## 注意事项

1. **端口固定为 9000** - 如果修改端口，需要同步修改配置
2. **仅支持 HTTP** - 如果使用 HTTPS，需要手动添加 HTTPS 变体
3. **C 段限制** - 只允许同一 C 段（如 192.168.158.x），不同 C 段需要重新配置

## 故障排查

如果仍然遇到 CSRF 错误：

1. **检查 .env 配置**
   ```bash
   cat .env | grep SENTRY_URL_PREFIX
   ```

2. **重启服务**
   ```bash
   docker compose restart sentry-web
   ```

3. **清除浏览器 Cookie**
   - 清除 Sentry 域名下的所有 Cookie
   - 重新登录

4. **查看日志**
   ```bash
   docker compose logs sentry-web --tail=50
   ```
